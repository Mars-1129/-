# TikStream AI 整体测试用例文档

## 1. 文档概述

### 1.1 文档名称
TikStream AI——电商场景 AIGC 带货视频生成系统整体测试用例文档

### 1.2 文档目标
本文档用于统一 TikStream AI 的测试范围、测试策略、测试数据、测试步骤与验收口径，覆盖素材、剧本、创作、分析看板及公共能力相关的功能测试、接口测试、AI/Agent 能力测试、边界测试、异常测试、权限/上下文测试与数据一致性测试。

### 1.4 测试原则
1. 主链路优先，先保通素材→剧本→创作→分析闭环。
2. P0 必测，P1 覆盖核心增强能力，P2 覆盖亮点和演示增强项。
3. 所有测试点必须绑定需求来源、接口路径、数据对象或状态机阶段。
4. 长任务类测试必须验证 `task_id`、`trace_id`、SSE、状态回收与失败原因。
5. 数据测试必须验证 PostgreSQL、Qdrant、MinIO、Redis/BullMQ 的一致性。

## 5. 端到端主链路测试用例

### TC-E2E-001 素材到成片闭环
- 模块：端到端
- 优先级：P0
- 需求来源：PRD 主流程、架构数据流、API 总纲
- 前置条件：存在有效 `product_id`，素材服务和任务服务正常
- 步骤：
  1. 上传一张商品主图和一段视频素材。
  2. 触发素材处理与切片。
  3. 使用快速模式生成剧本。
  4. 在分镜编辑器中保存剧本。
  5. 创建一键成片任务。
  6. 订阅 SSE 并等待任务完成。
  7. 打开成片详情并检查导出结果。
- 预期结果：
  - 素材入库成功
  - 切片状态进入完成态
  - 剧本输出结构化分镜
  - 创作任务状态可追踪并最终完成
  - 导出文件可访问
  - 任务页与看板可查看结果
- 核验点：`task_id`、`trace_id`、`video_url`、`current_stage`、数据库记录、对象存储文件

### TC-E2E-002 刷新恢复观察
- 优先级：P0
- 步骤：创建长任务后刷新页面，重新进入任务页并查询任务详情。
- 预期结果：可恢复当前状态、阶段、进度和失败信息。

## 6. 素材模块测试用例

### TC-MAT-001 图片上传成功
- 优先级：P0
- 输入：JPG/PNG/WebP，大小 ≤10MB
- 预期：生成 `Material` 记录和缩略图。

### TC-MAT-002 视频上传成功
- 优先级：P0
- 输入：MP4，大小 ≤200MB
- 预期：入库成功并触发后续处理。

### TC-MAT-003 上传时缺少 product_id
- 优先级：P0
- 预期：请求被拒绝，返回对应错误码。

### TC-MAT-004 超大文件拒绝
- 优先级：P0
- 输入：图片 >10MB 或视频 >200MB
- 预期：上传失败，不创建成功态业务记录。

### TC-MAT-005 切片时长约束
- 优先级：P0
- 预期：切片时长落在 1.5s–4.0s；过短片段舍弃，过长片段强制切分。

### TC-MAT-006 Dense Caption 生成
- 优先级：P0
- 预期：切片生成语义描述并落库。

### TC-MAT-007 Qdrant 向量写入
- 优先级：P1
- 预期：切片 `slice_id` 与向量库 point 一致。

### TC-MAT-008 同商品检索
- 优先级：P1
- 预期：检索结果只返回同 `product_id` 范围内切片。

### TC-MAT-009 重处理失败素材
- 优先级：P1
- 预期：失败素材可重新入队，状态正确更新。

## 7. 剧本模块测试用例

### TC-SCR-001 快速模式生成
- 优先级：P0
- 步骤：输入商品卖点、受众、风格、画幅，调用快速生成接口。
- 预期：返回结构化 `Script` 和 `ScriptShot[]`。

### TC-SCR-002 爆款仿写生成
- 优先级：P1
- 预期：基于参考视频结构输出新商品剧本，不返回原版权内容本体。

### TC-SCR-003 模板模式生成
- 优先级：P1
- 预期：使用模板策略与商品信息输出定制剧本。

### TC-SCR-004 剧本结构强校验
- 优先级：P0
- 预期：缺字段、歧义字段或不满足 schema 时保存失败。

### TC-SCR-005 总时长超限
- 优先级：P0
- 输入：总时长 >15 秒
- 预期：被拦截，不允许保存。

### TC-SCR-006 单分镜时长超限
- 优先级：P0
- 输入：单分镜 <1.5s 或 >5.0s
- 预期：保存失败并提示修改。

### TC-SCR-007 JSON Patch 局部更新
- 优先级：P1
- 预期：仅修改目标字段，其他字段保持不变。

### TC-SCR-008 音节配时校验
- 优先级：P1
- 预期：台词过长时返回校验失败与整改建议。

### TC-SCR-009 合规词法拦截
- 优先级：P1/P2
- 预期：绝对化、违规表述被阻断。

## 14. 数据一致性测试

### TC-DATA-001 材料与切片一致性
- 检查 `materials`、`material_slices`、Qdrant point 是否一一对应。

### TC-DATA-002 剧本与分镜一致性
- 检查 `scripts`、`script_shots`、接口返回是否一致。

### TC-DATA-003 创作任务一致性
- 检查 `creations`、`shot_renders`、SSE 事件与导出结果一致。

### TC-DATA-004 对象存储一致性
- 检查 `origin_url`、`stream_url`、`render_path`、`video_url` 可访问。

### TC-DATA-005 trace 一致性
- 检查 `trace_id` 在请求、日志、任务与回调中一致。
