# TikStream AI 素材模块接口文档

## 1. 文档目标与适用范围

### 1.1 文档名称
TikStream AI——电商场景 AIGC 带货视频生成系统素材模块接口文档

### 1.2 文档目标
本文档用于细化 TikStream AI 素材模块的接口设计，统一素材上传、素材列表、素材详情、素材删除、素材检索、素材重处理以及切片与向量化内部回调的接口契约，作为前端素材工作台联调、NestJS Controller / DTO 定义、上传链路实现、GPU Worker 回写和答辩复核的统一依据。

### 1.3 适用范围
本文档覆盖以下素材域能力：
- 素材上传与商品绑定
- 素材列表与详情查询
- 素材删除
- 切片语义检索与混合检索
- 素材重处理
- 切片处理回调
- 向量化结果回调
- 素材链路校验、权限、限流、错误码与异步承接说明

### 1.4 文档定位
本文档是 [全局API接口文档.md](全局API接口文档.md) 的素材域细化分册。全局请求 / 响应模型、统一错误码体系、统一 Header 规范、统一分页规范和全局安全边界以全局 API 文档为准；本文档只补充素材模块独有的接口字段、校验规则、回调契约和业务约束，不重复发明新的全局协议。

## 3. 素材域对象模型与状态枚举

### 3.1 Material 对象
Material 表示商家上传并绑定到某个商品上下文下的原始素材主记录，可为图片、视频或参考素材。

#### 3.1.1 Material 核心字段
| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| material_id | string(UUID) | 是 | 素材主键，对应 `materials.id` |
| product_id | string(UUID) | 是 | 所属商品主键 |
| file_name | string | 是 | 原始文件名 |
| type | string | 是 | `IMAGE` / `VIDEO` |
| source_type | string | 是 | `UPLOAD` / `REFERENCE` / `GENERATED`。`GENERATED` 由系统链路自动产生，不通过上传接口传入 |
| origin_url | string | 是 | 原始素材对象引用地址 |
| thumbnail_url | string | 否 | 缩略图地址 |
| file_size_bytes | integer | 是 | 文件大小 |
| duration_seconds | number | 否 | 视频时长，图片可为空 |
| width | integer | 否 | 媒体宽度 |
| height | integer | 否 | 媒体高度 |
| mime_type | string | 否 | MIME 类型 |
| status | string | 是 | `PENDING` / `PROCESSING` / `COMPLETED` / `FAILED` |
| slices_count | integer | 是 | 当前已生成切片数 |
| remark | string | 否 | 备注 |
| created_at | string | 是 | 创建时间 |
| updated_at | string | 是 | 更新时间 |

#### 3.1.2 source_type 说明
- `UPLOAD`：商家直接上传的原始素材。
- `REFERENCE`：外部引用或人工指定的参考素材。
- `GENERATED`：由系统创作链路或分析回写链路生成、回灌的派生素材。

### 3.2 MaterialSlice 对象
MaterialSlice 表示某条视频素材拆分后的原子切片记录，用于下游剧本召回、创作匹配和音效复用。

#### 3.2.1 MaterialSlice 核心字段
| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| id | string(UUID) | 是 | 切片主键，对应 `material_slices.id` |
| material_id | string(UUID) | 是 | 所属素材主键 |
| slice_id | string | 是 | 切片稳定映射键，对应 `material_slices.slice_id` |
| start_time | number | 是 | 开始时间，单位秒 |
| end_time | number | 是 | 结束时间，单位秒 |
| duration | number | 是 | 切片时长，单位秒 |
| dense_caption | string | 否 | 切片语义描述 |
| tags | array | 否 | 标签数组 |
| stream_url | string | 否 | 切片流地址 |
| key_frame_url | string | 否 | 关键帧地址 |
| embedding_version | string | 否 | 向量模型版本 |
| sfx_url | string | 否 | 提取音效对象地址 |
| status | string | 是 | `PENDING` / `CAPTIONING` / `EMBEDDING` / `COMPLETED` / `FAILED` |
| created_at | string | 是 | 创建时间 |
| updated_at | string | 是 | 更新时间 |

### 3.3 状态枚举

#### 3.3.1 MaterialStatus
- `PENDING`
- `PROCESSING`
- `COMPLETED`
- `FAILED`

#### 3.3.2 MaterialSliceStatus
- `PENDING`
- `CAPTIONING`
- `EMBEDDING`
- `COMPLETED`
- `FAILED`

### 3.4 状态流转说明

#### 3.4.1 素材状态流转
```text
PENDING → PROCESSING → COMPLETED / FAILED
```

#### 3.4.2 切片状态流转
```text
PENDING → CAPTIONING → EMBEDDING → COMPLETED / FAILED
```

### 3.5 关键标识约束
- `product_id`：素材访问、检索与权限控制的商品上下文边界。
- `material_id`：素材主资源 ID。
- `slice_id`：切片在 PostgreSQL 和向量检索侧的稳定映射键。
- `task_id`：重处理等异步任务对外追踪键。
- `trace_id`：上传、切片回写、向量化回写与日志排查的全链路追踪键。

## 4. 素材域外部接口清单总览

### 4.1 接口清单
| 方法 | 路径 | 优先级 | 调用方 | 说明 |
|---|---|---|---|---|
| POST | /api/v1/materials/upload | P0 | 前端素材工作台 | 上传素材并绑定商品 |
| GET | /api/v1/materials | P0 | 前端素材工作台 | 查询素材列表 |
| GET | /api/v1/materials/{material_id} | P0 | 前端素材工作台 / 创作工作台 | 查询素材详情与切片摘要 |
| DELETE | /api/v1/materials/{material_id} | P0 | 前端素材工作台 | 删除素材 |
| POST | /api/v1/materials/search | P1 | 创作工作台 / 剧本编辑器 / 素材工作台 | 混合检索素材切片 |
| POST | /api/v1/materials/{material_id}/reprocess | P1 | 前端素材工作台 | 重新触发素材处理 |

### 4.2 页面动作映射
- `/materials` 页面拖拽或点击上传：调用 `POST /api/v1/materials/upload`
- `/materials` 页面瀑布流与筛选：调用 `GET /api/v1/materials`
- `/materials` 页面打开素材详情抽屉：调用 `GET /api/v1/materials/{material_id}`
- `/materials` 页面删除素材：调用 `DELETE /api/v1/materials/{material_id}`
- `/materials` 页面或 `/create` 页面按语义词检索素材：调用 `POST /api/v1/materials/search`
- `/materials` 页面重新处理失败素材：调用 `POST /api/v1/materials/{material_id}/reprocess`

### 4.3 P0 / P1 / P2 边界说明
- P0：上传、列表、详情、删除
- P1：混合检索、重处理
- P2：9:16 自适应裁切、SFX 提取作为能力说明存在，但当前不单独扩展为新的对外素材接口

## 5. 素材上传接口

### 5.1 接口定义
- 方法：`POST`
- 路径：`/api/v1/materials/upload`
- Content-Type：`multipart/form-data`
- 作用：上传商品图片、视频或参考素材，并创建 Material 主记录
- 优先级：P0
- 调用方：前端素材工作台

### 5.2 请求 Header
| Header | 必填 | 说明 |
|---|---:|---|
| Content-Type | 是 | `multipart/form-data` |
| Authorization | 否 | 当前竞赛版本可选 |
| X-Trace-Id | 否 | 未传则后端生成 |
| X-Request-Id | 否 | 单次请求标识 |
| Idempotency-Key | 否 | 批量上传建议传入 |

### 5.3 表单字段
| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| product_id | string(UUID) | 是 | 所属商品 ID |
| file | binary | 是 | 上传素材文件 |
| type | string | 是 | `IMAGE` / `VIDEO` |
| source_type | string | 否 | `UPLOAD` / `REFERENCE`，默认 `UPLOAD`。注：`GENERATED` 由系统链路自动产生，不通过上传接口传入 |
| remark | string | 否 | 素材备注 |

### 5.4 上传规则
1. 上传时必须绑定有效 `product_id`。
2. 单次批量上传最多 10 个文件。
3. 前端分片传输标准为 5MB / chunk。
4. 前端并发上传建议限制为 3。
5. 必须支持断点续传。
6. 视频格式支持 `MP4`，编码以 H.264 High Profile 为推荐口径。
7. 图片格式支持 `JPG`、`PNG`、`WebP`。
8. 视频大小不得超过 200MB。
9. 图片大小不得超过 10MB。
10. 上传成功后应进入异步素材处理链路，而不是同步等待切片完成。
11. 批量上传时，接口返回结果数组 `materials[]`，每条元素结构与单文件上传成功响应一致；任一文件校验失败不影响其他文件的上传受理。

### 5.5 服务端处理语义
1. 校验 `product_id`、文件格式、大小和上传权限。
2. 原始文件落对象存储。
3. 写入 `materials` 主记录，初始状态为 `PENDING` 或 `PROCESSING`。
4. 图片与视频素材生成缩略图引用。
5. 派发后续素材入库异步链路，等待切片和打标结果回写。

### 5.6 成功响应字段
| 字段 | 类型 | 说明 |
|---|---|---|
| material_id | string(UUID) | 新创建素材主键 |
| product_id | string(UUID) | 商品主键 |
| file_name | string | 原始文件名 |
| type | string | `IMAGE` / `VIDEO` |
| source_type | string | 来源类型 |
| status | string | 初始状态 |
| thumbnail_url | string | 如已生成则返回 |
| file_size_bytes | integer | 文件大小 |
| created_at | string | 创建时间 |

### 5.9 主要错误码
- `PRODUCT_ID_REQUIRED`
- `FILE_FORMAT_NOT_SUPPORTED`
- `FILE_SIZE_EXCEEDED`
- `INVALID_REQUEST`
- `RATE_LIMITED`
- `OBJECT_STORAGE_WRITE_FAILED`
- `INTERNAL_SERVER_ERROR`

### 5.10 权限与限流
- 权限：仅允许在当前商品上下文内上传素材。
- 限流：按用户 / 商品维度限频，防止短时间并发上传压垮切片链路。
- 幂等：同一批上传请求建议传 `Idempotency-Key`，重复请求避免重复落主记录。

### 5.11 关联实体
- `products`
- `materials`

## 6. 素材列表接口

### 6.1 接口定义
- 方法：`GET`
- 路径：`/api/v1/materials`
- 作用：按商品上下文分页查询素材列表
- 优先级：P0

### 6.2 Query 参数
| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| product_id | string(UUID) | 是 | 商品 ID |
| status | string | 否 | `PENDING` / `PROCESSING` / `COMPLETED` / `FAILED` |
| type | string | 否 | `IMAGE` / `VIDEO` |
| page | integer | 否 | 页码，默认 1 |
| page_size | integer | 否 | 每页数量，默认 20 |
| sort_by | string | 否 | 推荐支持 `created_at` |
| sort_order | string | 否 | `asc` / `desc` |

### 6.3 查询语义
1. 查询必须在 `product_id` 范围内完成。
2. 支持按状态筛选处理中、完成、失败素材。
3. 支持按素材类型筛选图片与视频。
4. 支持按创建时间倒序显示最新素材。
5. 列表接口用于 `/materials` 瀑布流列表与基础筛选，不负责返回完整切片详情。

### 6.4 返回字段
| 字段 | 类型 | 说明 |

| material_id | string(UUID) | 素材主键 |
| product_id | string(UUID) | 商品主键 |
| file_name | string | 文件名 |
| type | string | 素材类型 |
| source_type | string | 来源类型 |
| thumbnail_url | string | 缩略图 |
| file_size_bytes | integer | 文件大小 |
| duration_seconds | number | 视频时长 |
| mime_type | string | MIME 类型 |
| status | string | 当前素材状态 |
| slices_count | integer | 已生成切片数 |
| remark | string | 备注 |
| created_at | string | 创建时间 |
| updated_at | string | 更新时间 |

## 7. 素材详情接口

### 7.1 接口定义
- 方法：`GET`
- 路径：`/api/v1/materials/{material_id}`
- 作用：查询素材主记录及其切片摘要
- 优先级：P0

### 7.2 Path 参数
| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| material_id | string(UUID) | 是 | 素材主键 |

### 7.3 查询语义
1. 返回 Material 主记录。
2. 返回当前素材的切片摘要列表。
3. 切片结果仅在切片链路完成后可见。
4. 若素材仍处于 `PENDING` 或 `PROCESSING`，可返回空切片数组和处理中状态。

- `file_name`
- `type`
- `source_type`
- `origin_url`
- `thumbnail_url`
- `file_size_bytes`
- `duration_seconds`
- `width`
- `height`
- `mime_type`
- `status`
- `slices_count`
- `remark`
- `created_at`
- `updated_at`

#### 7.4.2 slices
每个切片至少返回：
- `id`
- `material_id`
- `slice_id`
- `start_time`
- `end_time`
- `duration`
- `dense_caption`
- `tags`
- `stream_url`
- `key_frame_url`
- `embedding_version`
- `sfx_url`
- `status`
- `created_at`
- `updated_at`

## 12. 校验规则、权限与限流规则

### 12.1 上传校验规则
1. `product_id` 必填。
2. 支持上传类型：`MP4`、`JPG`、`PNG`、`WebP`。
3. 视频最大 200MB。
4. 图片最大 10MB。
5. 单次最多上传 10 个文件。
6. 前端采用 5MB 分片。
7. 前端并发上传建议不超过 3。
8. 必须支持断点续传。

### 12.2 切片规则
1. 目标切片时长控制在 `1.5s ≤ t ≤ 4.0s`。
2. 单镜头大于 `5.0s` 时强制等距切为 `3.0s` 级别切片。
3. 小于 `1.5s` 的碎片直接舍弃，不进入下游创作链路。
4. 切片失败后允许重试或通过重处理接口重新入链。

### 12.3 检索规则
1. 所有检索必须在单商品 `product_id` 上下文内进行。
2. 支持结构化时长过滤。
3. 支持关键字兜底检索。
4. 检索结果只返回业务字段，不返回底层向量实现细节。

### 12.4 权限规则
1. 当前竞赛版本采用单商家轻量上下文，不设计复杂组织权限。
2. 用户只可访问当前商品上下文下素材。
3. 不允许跨 `product_id` 访问、删除或检索素材。
4. 内部回调接口只允许内部 Worker / 服务访问。

### 12.5 限流规则
1. 上传接口按用户 / 商品维度限频。
2. 重处理接口按用户 / 商品维度限频，防止重算风暴。
3. 检索接口采用基础查询限频，并防止高频语义检索压垮下游链路。
4. Dense Caption 所依赖的文本模型调用遵循全局 Doubao 令牌桶治理，控制在 80 RPM 范围内。
5. 外部模型命中 429 时，内部链路应采用指数退避和错峰重试，但不把底层重试细节暴露成对外 API 参数。

### 12.6 幂等规则
1. 上传接口建议支持 `Idempotency-Key`。
2. `slice-callback` 以 `material_id + slice_id` 为幂等基础。
3. `embedding-callback` 以 `slice_id` 为幂等基础。
4. 重处理接口应避免短时间重复创建同一素材的多个重算任务。

### 12.7 AI 模块专属接口与能力承接说明
素材模块当前不单独向前端公开新的“模型直连接口”，而是通过业务接口与内部回调承接 AI 能力结果：
1. Dense Caption 能力：由切片处理链路在内部执行，结果通过 `POST /api/internal/v1/materials/{material_id}/slice-callback` 回写到 `dense_caption`、`tags`、`status` 等字段。
2. 多模态向量化能力：由向量化链路在内部执行，结果通过 `POST /api/internal/v1/material-slices/{slice_id}/embedding-callback` 回写到 `embedding_version` 与切片状态。
3. 对外检索承接接口：前端和下游模块统一通过 `POST /api/v1/materials/search` 消费语义检索结果，而不是直接访问向量模型或向量库。
4. AI 资源治理约束：Dense Caption 所依赖的文本模型调用采用批量聚合策略，推荐按 5 帧聚合降低调用频次；命中外部模型 429 时采用 `2s → 4s → 8s` 指数退避与错峰重试。
5. 安全边界：不暴露模型端点、API Key、向量维度、模型部署方式和底层资源标识。

## 13. 错误码与失败处理规范

### 13.1 素材域复用错误码
| 错误码 | HTTP 状态码 | 适用接口 | 含义 | 可重试 |
|---|---:|---|---|---:|
| INVALID_REQUEST | 400 | 全部 | 请求字段格式错误或缺失 | 否 |
| PRODUCT_ID_REQUIRED | 400 | upload / search / list | 缺失 `product_id` | 否 |
| FILE_FORMAT_NOT_SUPPORTED | 400 | upload | 文件格式不支持 | 否 |
| FILE_SIZE_EXCEEDED | 400 | upload | 文件大小超限 | 否 |
| MATERIAL_NOT_FOUND | 404 | detail / delete / reprocess / slice-callback | 素材不存在 | 否 |
| TASK_STATUS_CONFLICT | 409 | delete / reprocess | 当前状态不允许操作 | 否 |
| RATE_LIMITED | 429 | upload / search / reprocess | 命中限流或模型配额限制 | 是 |
| VECTOR_SEARCH_FAILED | 502 | search | 检索链路失败 | 是 |
| OBJECT_STORAGE_WRITE_FAILED | 502 | upload | 对象存储写入失败 | 是 |
| INTERNAL_WORKER_CALLBACK_FAILED | 502 | internal callbacks | 内部回调处理失败 | 是 |
| INTERNAL_SERVER_ERROR | 500 | 全部 | 未分类内部错误 | 视情况 |

### 13.2 失败处理原则
1. 用户可修复错误应返回字段级原因，例如格式错误、大小超限、缺失 `product_id`。
2. 系统可恢复错误应标记 `retryable=true`，例如限流、对象存储抖动、检索链路临时失败。
3. 素材处理失败和向量化失败必须保留 `trace_id` 便于回溯。
4. 回调失败不得静默吞掉，必须记录失败码和失败原因。

## 15. 异步链路与相关 SSE / 任务承接说明

### 15.1 异步处理总览
素材模块本身采用“REST 接入 + 内部回调回写”的异步处理模式。前端上传成功后不等待切片、Dense Caption 和向量化同步完成，而是先得到素材主记录，然后由后端异步推进处理链路。

### 15.2 素材入库链路
```text
前端上传素材
  → 网关校验 product_id / 格式 / 大小 / 权限
  → 对象存储落原始文件
  → 创建 Material 主记录
  → 派发素材入库异步事件
  → Worker 执行切片
  → 关键帧与 Dense Caption 生成
  → slice-callback 回写 MaterialSlice 结果
  → 向量化执行
  → embedding-callback 回写 embedding_version 与切片状态
  → Material / MaterialSlice 最终进入 COMPLETED
```

### 15.3 与 SSE 的边界说明
1. 素材模块当前不单独定义新的前端 SSE 订阅接口。
2. 素材工作台主要通过轮询列表和详情查询感知素材处理状态。
3. 若未来需要统一长任务观察，可复用 [全局API接口文档.md](全局API接口文档.md) 中的任务与 SSE 机制，而不是在素材分册中另起协议。

### 15.4 与下游模块的承接关系
1. 创作模块通过 `POST /api/v1/materials/search` 消费素材切片检索能力。
2. 剧本分镜中的 `selected_slice_id` 最终依赖素材切片域提供的 `slice_id`。
3. 切片的 `stream_url`、`key_frame_url`、`sfx_url` 是创作模块可消费的关键结果字段。

## 16. 命名规范、版本约定与变更边界

### 16.1 URL 命名规范
1. 使用小写复数资源名：`/materials`。
2. 动作型接口使用资源后缀：`/reprocess`。
3. 内部回调统一放在 `/api/internal/v1` 前缀下。

### 16.2 字段命名规范
1. 对外 JSON 字段统一采用蛇形命名。
2. 字段命名必须与数据库设计文档保持一致。
3. 不得在素材文档中引入 `asset_id`、`clip_id` 等与正式文档冲突的别名。

### 16.3 枚举与错误码命名规范
1. 枚举值统一使用稳定英文大写或约定短语。
2. 错误码统一复用全局错误码体系。
3. 若新增素材域特定错误码，必须与全局 API 文档同步。

### 16.4 版本变更约定
1. 新增字段优先向后兼容。
2. 若删除字段或调整语义，必须升级版本或声明废弃策略。
3. 若新增素材能力接口，必须先与全局 API 文档对齐，再进入本分册。

本素材模块接口文档围绕 TikStream AI 的素材上传、素材管理、素材检索和内部回写链路，建立了统一的字段命名、状态枚举、校验规则、错误码、权限边界和异步处理契约，能够直接支撑前端素材工作台联调、NestJS DTO / Controller 设计、GPU Worker 回写实现以及竞赛答辩中的素材链路接口说明。同时，本文档与 [全局API接口文档.md](全局API接口文档.md)、[数据库设计文档.md](数据库设计文档.md)、[整体系统架构设计文档.md](整体系统架构设计文档.md) 保持一致，为后续继续拆分剧本模块接口文档、创作模块接口文档和分析看板接口文档提供统一接口口径。
