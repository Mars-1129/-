# TikStream AI 分析看板接口文档

## 1. 文档目标与适用范围

### 1.1 文档名称
TikStream AI——电商场景 AIGC 带货视频生成系统分析看板接口文档

### 1.2 文档目标
本文档用于细化 TikStream AI 分析看板模块的正式接口契约，统一前端 `/analytics` 页面、NestJS Analytics Controller / Service、DuckDB 预计算结果承接层、ECharts 5 图表消费层以及分析诊断回写创作链路之间的接口设计，作为前后端联调、DTO 定义、图表渲染、异常处理、测试验收与答辩复核的统一依据。

### 1.3 适用范围
本文档覆盖以下分析域能力：
- 分镜留存曲线查询
- 多因子归因热力图查询
- 视听留存桑基图查询
- A/B 版本效果对比
- 一键自愈诊断与回写
- 分析域对象模型、校验规则、权限规则、限流规则、错误码与示例报文
- Mock 数据、预计算边界与创作链路衔接说明

### 1.4 文档定位
本文档是 [全局API接口文档.md](全局API接口文档.md) 在分析域的细化分册。全局请求 / 响应模型、统一错误码体系、统一 Header 规范、统一权限与限流边界以全局 API 文档为准；本文档只补充分析域特有的对象模型、图表数据结构、参数校验、异常场景、自愈回写语义以及与 DuckDB 预计算结果之间的契约，不重复发明新的全局协议。

### 1.5 当前阶段能力边界
当前版本分析看板明确采用“Mock 数据 + DuckDB 预计算 + 前端可视化消费”的实现口径，边界如下：
1. 当前所有效果指标均为模拟值、预测值或预计算归因结果，不等价于真实广告平台官方回流数据。
2. 当前不开放真实投流平台回流采集、实时因果推断、组织级归因权限管理等复杂能力。
3. 当前分析域重点服务竞赛 MVP 演示闭环，优先保证留存曲线、热力图、桑基图、A/B 对比和一键自愈链路完整
4. 一键自愈是诊断结果回写创作链路的入口，不等价于自动投流调控服务。

### 1.6 不在本文档直接公开的内容
以下内容属于内部实现边界，不作为外部 API 字段或公开接口直接暴露：
1. DuckDB 预计算 SQL、Mock 数据生成脚本与内部诊断权重。
2. 分析诊断 Prompt、策略解释模板与内部提示词工程。
3. 预计算缓存失效策略、数据刷新调度与内部目录结构。
4. 创作自愈时的具体模型端点、底层 Prompt、局部重渲染内部编排细节。
5. 真实广告平台接入、埋点采集与数仓同步实现。

## 2. 与全局 API 规范的继承关系

### 2.1 Base URL 与版本继承
- Base URL：`/api`
- 当前主版本：`v1`
- 分析域对外完整路径前缀：`/api/v1`

### 2.2 通信协议继承
- 默认响应格式：`application/json`
- 传输协议：HTTPS
- 当前分析域不使用 `multipart/form-data` 或 `application/json-patch+json` 作为主协议
- 一键自愈使用标准 JSON 请求体

### 2.3 通用 Header 继承
| Header | 必填 | 说明 |
|---|---:|---|
| Authorization | 否 | 当前竞赛版本可选，预留 Bearer Token 口径 |
| X-Trace-Id | 否 | 调用方可传入链路追踪 ID，未传则后端生成 |
| X-Request-Id | 否 | 单次请求标识 |
| Idempotency-Key | 否 | 一键自愈等会触发异步回写的接口建议传入 |
| Content-Type | 是 | GET 接口省略，POST 接口为 `application/json` |

### 2.4 成功响应结构继承
分析域所有接口继承 [全局API接口文档.md](全局API接口文档.md) 的统一成功响应包裹：

```json
{
  "success": true,
  "message": "ok",
  "data": {},
  "trace_id": "trc_20260523_xxxx",
  "timestamp": "2026-05-23T10:30:00Z"
}
```

### 2.5 失败响应结构继承
分析域所有接口继承全局统一失败响应结构：

```json
{
  "success": false,
  "message": "validation failed",
  "error": {
    "code": "INVALID_REQUEST",
    "details": [
      {
        "field": "product_id",
        "reason": "product_id is required"
      }
    ],
    "retryable": false
  },
  "trace_id": "trc_20260523_xxxx",
  "timestamp": "2026-05-23T10:30:00Z"
}
```

### 2.6 分页规范继承
当前分析域主接口以单图表或单报表查询为主，不以分页列表为核心；若后续扩展历史分析快照列表等能力，统一继承全局分页参数：
- `page`
- `page_size`
- `sort_by`
- `sort_order`

### 2.7 命名继承与一致性来源
分析域字段命名、资源路径、错误码命名与主键口径必须与以下正式文档保持一致：
- [全局API接口文档.md](全局API接口文档.md)
- [产品需求文档.md](产品需求文档.md)
- [整体系统架构设计文档.md](整体系统架构设计文档.md)
- [创作模块接口文档.md](创作模块接口文档.md)

### 2.8 分析域特殊约束说明
1. 所有分析结果必须明确标记为 Mock / 预测 / 预计算结果。
2. 所有接口均以当前商品上下文为边界，禁止跨商品混查。
3. A/B 对比仅允许比较同一商品下的不同创作版本。
4. 一键自愈不新建独立任务协议，统一复用创作域 `creation_id`、`task_id` 与任务状态机。

## 3.8 唯一键与定位约束
1. `product_id` 是分析域权限和查询边界的第一层约束。
2. `creation_id` 是视频版本级分析的主定位键。
3. `comparison_creation_id` 仅用于 A/B 对比，不得替代 `creation_id` 的主语义。
4. `task_id` 仅在一键自愈触发创作任务后返回，复用创作域任务追踪口径。
5. `trace_id` 是分析查询、自愈回写、任务联调和日志排查的全链路追踪键。

### 4.1 主接口清单
| 方法 | 路径 | 优先级 | 调用方 | 说明 |
|---|---|---|---|---|
| GET | /api/v1/analytics/retention-curve | P1 | `/analytics` 页面 | 分镜留存曲线 |

### 4.2 页面动作映射
- `/analytics` 页面首次进入：优先加载留存曲线与看板摘要。
- `/analytics` 页面切换“多因子归因”标签：调用 `GET /api/v1/analytics/style-factors`。
- `/analytics` 页面切换“视听流向”标签：调用 `GET /api/v1/analytics/audio-visual-sankey`。
- `/analytics` 页面切换 A/B 对比视图：调用 `GET /api/v1/analytics/ab-compare`。
- 用户点击留存曲线下滑拐点或 A/B 异常诊断结果：可触发一键自愈入口。
- 用户点击“一键自愈”：调用 `POST /api/v1/analytics/self-heal`。

### 4.3 P0 / P1 / P2 边界说明
- P0：当前分析域无独立 P0 对外接口，但属于整体闭环的重要展示能力。
- P1：分镜留存曲线。
- P2：热力图、桑基图、A/B 对比、一键自愈。

## 5. 分镜留存曲线接口

### 5.1 接口定义
- 方法：`GET`
- 路径：`/api/v1/analytics/retention-curve`
- 作用：查询指定商品某个创作版本的分镜留存曲线、分镜切换点与下滑拐点
- 优先级：P1
- 调用方：分析看板页

### 5.2 Query 参数
| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `product_id` | string(UUID) | 是 | 商品主键 |

## 9. 一键自愈接口

### 9.1 接口定义
- 方法：`POST`
- 路径：`/api/v1/analytics/self-heal`
- Content-Type：`application/json`
- 作用：根据分析结果回写创作链路，生成自愈建议或触发局部优化任务
- 优先级：P2
- 调用方：分析看板页

### 9.2 请求体
| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `product_id` | string(UUID) | 是 | 商品主键 |
| `creation_id` | string(UUID) | 是 | 原始版本主键 |
| `trigger_source` | string | 是 | `RETENTION_DROP` / `AB_COMPARE` / `MANUAL` |
| `target_shot_indexes` | array(integer) | 否 | 受影响分镜序号 |
| `issue_type` | string | 是 | 问题类型 |
| `strategy` | string | 是 | 自愈策略 |
| `dry_run` | boolean | 否 | 是否仅返回建议，默认 `false` |
| `remark` | string | 否 | 人工补充说明 |

### 9.3 校验规则
1. `product_id`、`creation_id` 必须属于同一商品上下文。
2. `issue_type`、`strategy`、`trigger_source` 必须属于受控枚举。
3. `target_shot_indexes` 传入时必须属于当前版本分镜范围。

1. 校验请求上下文与目标版本。

3. 若为 `dry_run`，返回建议摘要与受影响分镜清单。
4. 若为正式执行，生成创作回写任务，复用 [创作模块接口文档.md](创作模块接口文档.md) 的任务状态机与 `task_id` 语义，并返回 `healed_creation_id`。
5. 正式执行入口返回 `status=QUEUED`，表示任务已进入创作链路排队阶段。
6. 如涉及局部分镜改写，优先复用已有脚本与创作链路，不新增平行流程。

## 10. 权限、限流与幂等规则

### 10.1 权限规则
当前竞赛版本采用轻量化单商家口径，不设计复杂组织权限。分析域默认规则如下：
1. 用户仅可访问当前商品上下文内的分析结果。
2. A/B 对比仅允许比较当前商品下的两个版本。
3. 一键自愈仅允许回写当前商品的创作链路。
4. 内部预计算脚本、诊断规则与缓存刷新接口不对前端开放。

### 10.2 限流规则
| 接口 | 规则 |
|---|---|
| `GET /analytics/retention-curve` | 按用户 / 商品维度限频，防止高频拖动时间范围造成重复查询 |
| `GET /analytics/style-factors` | 按用户维度限频，防止高频切换图表维度 |
| `GET /analytics/audio-visual-sankey` | 按用户维度限频，防止大图表重复刷新 |
| `GET /analytics/ab-compare` | 按商品维度限频，防止短时间重复比较多个版本 |
| `POST /analytics/self-heal` | 按商品和版本维度限频，防止重复创建回写任务 |

### 10.3 幂等规则
1. 所有 GET 接口天然幂等。
2. `POST /api/v1/analytics/self-heal` 建议传入 `Idempotency-Key`。
3. 同一 `creation_id + issue_type + strategy` 在短时间内重复提交时，应避免重复创建回写任务。

## 11. 错误码与失败处理规则

### 11.1 优先复用的全局错误码
分析域优先复用 [全局API接口文档.md](全局API接口文档.md) 的统一错误码：
- `INVALID_REQUEST`
- `TASK_NOT_FOUND`
- `TASK_STATUS_CONFLICT`
- `RATE_LIMITED`
- `INTERNAL_SERVER_ERROR`

### 11.2 分析域建议补充的业务错误码
若后续需要对分析域做更细化实现，建议补充以下错误码并同步回补到全局 API 文档：

| 错误码 | HTTP 状态码 | 含义 | 可重试 |
|---|---:|---|---:|
| `ANALYTICS_DATA_NOT_READY` | 409 | 预计算数据尚未生成完成 | 是 |
| `ANALYTICS_DIMENSION_UNSUPPORTED` | 400 | 分析维度参数不受支持 | 否 |
| `AB_COMPARE_INCOMPLETE` | 422 | A/B 数据不完整，仅可部分展示 | 否 |

### 11.3 失败处理原则
1. 用户可修复问题返回字段级提示，不返回笼统报错。
2. 数据缺失优先返回可展示的兜底结构或明确提示，不直接返回空白页面语义。
3. 一键自愈冲突时返回可追踪的 `task_id` 或冲突原因。
4. 所有错误响应都必须附带 `trace_id`。

## 12. 数据一致性与实现边界

### 12.1 与 PRD 的一致性要求
分析域接口必须完整覆盖下列需求：
- FR-20：多因子归因热力图
- FR-21：视听留存桑基图
- FR-22：分镜留存曲线
- FR-23：A/B 对比
- FR-24：一键自愈

### 12.2 与全局 API 的一致性要求
1. 路径必须使用 `/api/v1/analytics/*`，与全局 API 版本前缀保持一致。
2. 成功 / 失败响应结构必须复用全局包裹结构。
3. 权限、限流、幂等、错误码优先继承全局文档。
4. `task_id`、`creation_id`、`trace_id` 命名必须与创作域一致。

### 12.3 与前端图表消费的一致性要求
1. 热力图返回结构必须适配 ECharts Heatmap 所需的轴标签与单元格数据。
2. 桑基图返回结构必须包含稳定 `nodes` / `links`。
3. 留存曲线必须包含坐标点与分镜标记，支撑拐点高亮。
4. A/B 对比必须能支撑卡片化差异展示与诊断摘要渲染。

### 12.4 Mock / 预计算边界说明
1. 当前所有结果基于 Mock 数据和 DuckDB 预计算结果。
2. 必须显式返回 `data_source`、`is_mock`、`is_predicted` 等字段。
3. 当前不宣称真实投流平台回流结果或真实因果推断结论。
4. 数据缺失时应返回兜底提示，契合 PRD 的异常场景定义。

### 12.5 与创作链路的边界说明
1. 分析模块只负责诊断与建议，不负责独立渲染协议。
2. 一键自愈触发后统一复用创作模块任务体系。
3. 局部分镜重写、重渲染、生成新版本等能力，统一遵循 [创作模块接口文档.md](创作模块接口文档.md) 语义。
