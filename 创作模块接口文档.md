# TikStream AI 创作模块接口文档

## 1. 文档目标与适用范围

### 1.1 文档名称
TikStream AI——电商场景 AIGC 带货视频生成系统创作模块接口文档

### 1.2 文档目标
本文档用于定义 TikStream AI 创作模块的正式接口契约，统一前端创作工作台、NestJS Creation Controller / DTO、BullMQ 长任务编排、SSE 事件推送、Remotion 预览编排、正式导出流程、分镜级局部重渲染、素材切片替换、失败重试、任务取消以及内部 Worker 回调协同的接口设计，作为前后端联调、后端实现、测试验收与答辩复核的统一依据。

### 1.3 适用范围
本文档是 [全局API接口文档.md](全局API接口文档.md) 在创作域的细化分册，覆盖以下范围：
- 一键成片接口
- 创作结果查询接口
- 预览态编排结果查询接口
- 成片导出接口
- 局部分镜重渲染接口
- 替换素材切片接口
- 失败任务重试接口
- 主动取消任务接口
- 任务列表 / 任务详情 / SSE 事件订阅接口
- 内部阶段回写、导出完成回写、失败补偿回写接口
- 创作域的权限、限流、幂等、错误码、示例报文与能力边界说明

### 1.4 文档定位
当前版本采用“全局总纲接口文档 + 按模块拆分子接口文档”的组织方式。本文档不重复发明全局协议，而是在已完成的 [全局API接口文档.md](全局API接口文档.md) 基础上，对创作域接口的字段、校验、状态机、SSE 载荷、内部回调边界和实现承接方式进行完整细化。

### 1.5 不在本文档直接公开的内容
以下内容属于内部实现边界，不作为外部 API 字段或公开接口直接暴露：
1. Prompt 文本、Few-Shot 样例、上下文压缩策略。
2. 图生视频底层调用参数、模型端点、密钥和资源配置。
3. F5-TTS 内部超参数、字幕对齐内部算法、BGM Ducking 具体滤波参数。
4. FFmpeg 分镜拼接与 loudnorm 重编码命令。
5. BullMQ Job 内部结构、队列命名、Redis 锁细节、SceneHash / MD5 计算实现。
6. MinIO 内部目录结构、缓存对象路径、临时资源地址与访问凭据。

## 2. 与全局 API 规范的继承关系

### 2.1 Base URL 与版本继承
创作模块接口继承全局 API 版本规范：
- 外部业务接口前缀：`/api/v1`
- 内部回调接口前缀：`/api/internal/v1`

### 2.2 通信协议继承
- 默认响应格式：`application/json`
- SSE 订阅：`text/event-stream`
- 当前创作模块不使用 `application/json-patch+json` 作为对外主协议；分镜编辑后的创作干预结果通过专门业务接口承接

### 2.3 通用 Header 继承
| Header | 必填 | 说明 |
|---|---:|---|
| Authorization | 否 | 当前竞赛版本可选，预留 Bearer Token 口径 |
| X-Trace-Id | 否 | 调用方传入链路追踪 ID，未传则由网关生成 |
| X-Request-Id | 否 | 单次请求标识 |
| Idempotency-Key | 否 | 一键成片、导出、重试、内部回调等幂等接口建议传入 |
| Content-Type | 是 | 与接口类型匹配，JSON 接口为 `application/json` |

### 2.4 成功响应结构继承
创作模块普通 JSON 接口统一继承全局成功响应结构：

```json
{
  "success": true,
  "message": "ok",
  "data": {},
  "trace_id": "trc_20260523_xxxx",
  "timestamp": "2026-05-23T10:30:00Z"
}
```

### 2.5 分页响应结构继承
任务列表接口继承全局分页响应结构：

```json
{
  "success": true,
  "message": "ok",
  "data": {
    "items": [],
    "page": 1,
    "page_size": 20,
    "total": 1,
    "has_more": false
  },
  "trace_id": "trc_20260523_xxxx",
  "timestamp": "2026-05-23T10:30:00Z"
}
```

### 2.6 失败响应结构继承
创作模块错误响应统一继承全局失败结构：

```json
{
  "success": false,
  "message": "validation failed",
  "error": {
    "code": "TASK_STATUS_CONFLICT",
    "details": [
      {
        "field": "creation_id",
        "reason": "current task stage does not allow this operation"
      }
    ],
    "retryable": false
  },
  "trace_id": "trc_20260523_xxxx",
  "timestamp": "2026-05-23T10:30:00Z"
}
```

### 2.7 SSE 结构继承
创作模块 SSE 接口继承全局事件结构：

```text
event: task.stage.changed
id: tsk_20260523_xxxx
data: {"task_id":"tsk_20260523_xxxx","status":"PROCESSING","current_stage":"TTS_GENERATING","progress":62,"message":"voiceover generation completed","trace_id":"trc_20260523_xxxx","timestamp":"2026-05-23T10:35:12Z"}
```

### 2.8 创作模块与全局规范的特殊关系说明
1. 创作模块同时包含普通 JSON 接口与长连接 SSE 接口。
2. 一键成片、导出、重试本质上都是长任务触发接口，返回的是任务受理结果而不是同步成片结果。
3. 创作模块必须严格遵循全局定义的路径、字段、错误码和事件名，统一采用 `/api/v1/creations/*`、`/api/v1/tasks/*` 等命名规范，不使用 `progress_update` 等非标准事件名。

### 3.9 关键定位与唯一性约束
1. `creation_id` 是创作记录主键。
2. `task_id` 是外部长任务追踪主键。
3. `trace_id` 是网关、Worker、导出和 SSE 的全链路追踪键。
4. `script_shot_id` 是局部分镜重渲染与换片的稳定定位键。
5. `script_id + shot_index` 是剧本分镜稳定排序与冗余校验组合键。
6. `slice_id`（ShotRender 侧）与 `selected_slice_id`（换片请求参数侧）是创作域承接素材切片绑定关系的标准字段，前者落库使用 `slice_id`，后者为业务语义字段。

## 5. 一键成片接口

### 5.1 接口定义
- 方法：`POST`
- 路径：`/api/v1/creations`
- 作用：根据商品与已生成剧本创建一条创作长任务，进入创作状态机主链路
- 权限：当前商品上下文内可写
- 限流：防止同商品 / 同剧本短时间重复创建任务风暴
- 幂等：建议传入 `Idempotency-Key`

### 5.2 业务说明
该接口用于创建一条正式的创作任务，而不是直接同步返回成片结果。创建成功后，后端写入 `creations` 主记录，生成 `task_id`，并将任务推入 BullMQ，由状态机从 `QUEUE_ALLOCATION` 阶段开始推进。

### 5.3 请求参数

#### 5.3.1 Request Body
| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `product_id` | string(UUID) | 是 | 商品主键 |
| `script_id` | string(UUID) | 是 | 剧本主键 |
| `engine_mode` | string | 否 | 创作模式，默认 `SCRIPT_DRIVEN` |
| `target_resolution` | string | 否 | 目标分辨率，默认 `1080x1920` |
| `export_format` | string | 否 | 导出格式（容器+编码），默认 `MP4`（H.264 编码） |
| `voice_profile` | string | 否 | 配音音色标识 |
| `bgm_policy` | string | 否 | BGM 策略标识 |
| `force_refresh` | boolean | 否 | 是否忽略既有分镜渲染缓存，默认 `false` |

### 5.4 校验规则
1. `product_id` 必须存在。
2. `script_id` 必须存在。
3. 剧本必须属于当前商品上下文。
4. 剧本结构必须通过强校验，不得存在缺失分镜、缺失字幕、缺失安全区等结构性问题。
5. 剧本总时长不得超过 15 秒。
6. 每个分镜时长必须满足 `1.5s~5.0s`。
7. 若分镜存在已绑定切片，则切片时长需满足 `1.5s~4.0s`。当分镜时长超出单切片时长范围时，系统可拼接多个切片或触发图生视频链路补足差值。
8. 若前一个同剧本创作任务仍处于不允许并行的处理中状态，可直接返回状态冲突。

### 5.5 请求示例
```json
{
  "product_id": "8cf8fe7f-9eb8-4b80-9e28-8f94a83f7611",
  "script_id": "cc65b66b-6288-49a6-88f4-9980ff2e4d36",
  "engine_mode": "SCRIPT_DRIVEN",
  "target_resolution": "1080x1920",
  "export_format": "MP4",
  "voice_profile": "female_cn_young_01",
  "bgm_policy": "AUTO_MATCH",
  "force_refresh": false
}
```

### 5.6 成功响应示例
```json
{
  "success": true,
  "message": "task created",
  "data": {
    "creation_id": "655b0c66-168c-46cc-a763-c10cae8c6ee8",
    "task_id": "tsk_20260523_000001",
    "product_id": "8cf8fe7f-9eb8-4b80-9e28-8f94a83f7611",
    "script_id": "cc65b66b-6288-49a6-88f4-9980ff2e4d36",
    "status": "PENDING",
    "current_stage": "QUEUE_ALLOCATION",
    "progress": 0
  },
  "trace_id": "trc_20260523_000001",
  "timestamp": "2026-05-23T10:00:00Z"
}
```

### 5.7 响应字段说明
| 字段 | 类型 | 说明 |
|---|---|---|
| `creation_id` | string(UUID) | 创作主键 |
| `task_id` | string | 任务主键 |
| `product_id` | string(UUID) | 商品主键 |
| `script_id` | string(UUID) | 剧本主键 |
| `status` | enum | 初始状态，通常为 `PENDING` |
| `current_stage` | enum | 初始阶段，固定为 `QUEUE_ALLOCATION` |
| `progress` | integer | 初始进度，默认 0 |

### 5.8 可能错误码
| 错误码 | HTTP 状态码 | 含义 | 可重试 |
|---|---:|---|---:|
| `INVALID_REQUEST` | 400 | 请求参数格式错误 | 否 |
| `PRODUCT_ID_REQUIRED` | 400 | 缺失 `product_id` | 否 |
| `SCRIPT_NOT_FOUND` | 404 | 剧本不存在 | 否 |
| `SCRIPT_SCHEMA_INVALID` | 422 | 剧本结构非法 | 否 |
| `SCRIPT_DURATION_EXCEEDED` | 422 | 总时长或分镜时长不合法 | 否 |
| `COMPLIANCE_CHECK_FAILED` | 422 | 剧本文案命中合规拦截 | 否 |
| `TASK_STATUS_CONFLICT` | 409 | 当前商品 / 剧本已有冲突中的任务 | 否 |
| `IDEMPOTENCY_CONFLICT` | 409 | 幂等键冲突 | 视情况 |
| `RATE_LIMITED` | 429 | 命中创建限流规则 | 是 |
| `MODEL_PROVIDER_FAILED` | 502 | 依赖能力准备或预热失败 | 是 |
| `INTERNAL_SERVER_ERROR` | 500 | 系统内部错误 | 视情况 |

## 8. 导出成片接口

### 8.1 接口定义
- 方法：`POST`
- 路径：`/api/v1/creations/{creation_id}/export`
- 作用：触发正式成片导出任务
- 权限：当前商品上下文内可写
- 幂等：建议传入 `Idempotency-Key`

### 8.2 业务说明
当前系统采用前后端同构混合渲染设计：
- 预览阶段：前端 Remotion Player 在浏览器中完成高还原度预览。
- 导出阶段：服务端 Remotion Headless + Chromium + FFmpeg 完成正式导出。

因此，导出接口与预览接口职责不同，不得混用。

### 8.3 请求参数
| 字段 | 类型 | 必填 | 说明 |

| `codec` | string | 否 | 默认 `H264` |
| `format` | string | 否 | 默认 `MP4` |
| `force_recompose` | boolean | 否 | 是否强制重新拼接，默认 `false` |

### 8.4 校验规则
1. 当前版本正式导出默认标准为 `1080x1920`、`30fps`、`H.264`、`MP4`。
2. 仅允许对可导出的创作记录触发导出。
3. 若必要前置阶段尚未完成，应拒绝导出或进入串行等待逻辑。
4. 正式导出前必须经过 `LOUDNORM_COMPLIANCE` 阶段，以满足响度规范。

## 9. 局部分镜重渲染接口

### 9.1 接口定义
- 方法：`POST`
- 路径：`/api/v1/creations/{creation_id}/rerender-shot`
- 作用：对单个分镜进行局部重渲染，而不重新渲染整片
- 权限：当前商品上下文内可写

### 9.2 业务说明
当用户在编辑器中修改分镜台词、调整时长、替换素材或变更局部风格因子时，不应强制全片重渲染。系统通过 `script_shot_id` 定位目标分镜，并基于缓存命中情况仅重算受影响分镜，再进行增量拼接。

| `reason` | string | 否 | 重渲染原因说明 |

### 9.4 校验规则
1. `script_shot_id` 必须存在且属于当前创作关联剧本。
2. `shot_index` 必须与 `script_shot_id` 对应关系一致。
3. 若目标分镜当前处于不可中断原子阶段，可延迟处理或返回状态冲突。

### 13.3 SSE 事件订阅接口

#### 13.3.1 接口定义
- 方法：`GET`
- 路径：`/api/v1/tasks/{task_id}/events`
- 响应类型：`text/event-stream`
- 作用：实时订阅任务阶段变化与进度事件

#### 13.3.2 事件类型
创作域必须沿用全局事件类型，不使用旧版 `progress_update`、`task_completed`、`task_error`：

```text
task.created
task.stage.changed
task.progress.updated
task.completed
task.failed
task.canceled
shot.render.completed
shot.render.failed
heartbeat
```

#### 13.3.3 SSE Data 字段
| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `task_id` | string | 是 | 任务主键 |
| `status` | string | 是 | 当前状态 |
| `current_stage` | string | 是 | 当前阶段 |
| `progress` | integer | 是 | 当前进度 |
| `message` | string | 否 | 说明信息 |
| `trace_id` | string | 否 | 链路追踪 ID |
| `timestamp` | datetime | 是 | 事件时间 |

#### 13.3.4 SSE 示例
```text
event: task.progress.updated
id: tsk_20260523_000001
data: {"task_id":"tsk_20260523_000001","status":"PROCESSING","current_stage":"AI_VIDEO_GENERATING","progress":47,"message":"第 3 个分镜生成完成","trace_id":"trc_20260523_000001","timestamp":"2026-05-23T10:02:10Z"}
```

#### 13.3.5 业务规则
1. 前端长任务观察默认优先使用 SSE，轮询仅作兜底。
2. 页面刷新或重新进入后，应先通过任务详情接口恢复当前状态，再重新建立 SSE 连接。
3. 同一 `task_id` 的 SSE 连接数必须受限。
4. 服务端应提供心跳事件或等效保活机制，防止中间层断流。
5. 任务进入 `FINISHED`、`FAILED`、`CANCELED` 后，必须发送终态事件。

### 14.4 内部回调规则
1. 内部回调不对前端开放。
2. 内部回调必须支持幂等处理。

4. 回调成功后，应同步刷新任务详情并触发 SSE 事件。
5. 回调层只承接结果，不向外暴露具体队列结构、缓存路径和模型资源信息。

## 15. 校验规则、权限与限流规则

### 15.1 创作前置校验规则
1. `product_id` 必须存在且有效。
2. `script_id` 必须存在且属于当前商品上下文。
3. 剧本必须通过结构化校验。
4. 总视频时长必须 `<= 15s`。
5. 单分镜时长必须在 `1.5s~5.0s` 之间。
6. 若分镜已绑定素材切片，则切片时长必须在 `1.5s~4.0s` 之间。
7. 分镜必须具备 `scene_description_query`、`voiceover_text`、`subtitle_text`、`safe_zone_bounding_box` 等核心字段。

### 15.2 预览与导出校验规则
1. 预览接口返回的是编排结果，不要求正式导出已完成。
2. 导出前必须保证关键阶段完成，并进入或完成 `LOUDNORM_COMPLIANCE`。
3. 导出默认仅支持 `1080x1920`、`30fps`、`H.264`、`MP4`。

### 15.3 局部干预校验规则
1. `script_shot_id` 与 `shot_index` 必须一致映射到同一分镜。
2. `selected_slice_id` 必须属于当前商品素材域。
3. 替换切片必须满足时长、可用性和消费约束。
4. 分镜级重渲染优先复用缓存，仅在 `force_refresh=true` 或缓存失效时重算。

### 15.4 权限规则
1. 用户仅能访问当前商品上下文内的创作任务。
2. 跨商品访问必须拒绝。
3. 内部回调仅允许内部 Worker 身份调用。

### 15.5 限流规则
| 场景 | 规则 |
|---|---|
| 一键成片 | 防止同商品 / 同剧本短时间重复创建任务 |
| 图生视频补足 | 受 Seedance 分布式信号量限制，并发上限 3 |
| SSE 订阅 | 同一任务允许有限连接数 |
| 重试接口 | 防止失败任务频繁重试造成风暴 |
| 导出接口 | 同一创作任务重复导出需串行控制 |

### 15.6 幂等规则
1. 一键成片建议传入 `Idempotency-Key`。
2. 导出、重试、取消接口建议支持幂等处理。
3. 内部回调必须以 `task_id` 或阶段语义作为幂等基础。
4. 导出完成回写重复上报时，不得重复生成最终业务结果。

## 16. 错误码与失败处理规范

### 16.1 创作域主要错误码清单
| 错误码 | HTTP 状态码 | 含义 | 可重试 |
|---|---:|---|---:|
| `INVALID_REQUEST` | 400 | 请求体格式错误或字段缺失 | 否 |
| `PRODUCT_ID_REQUIRED` | 400 | 创建请求缺失 `product_id` | 否 |
| `SCRIPT_NOT_FOUND` | 404 | 剧本不存在 | 否 |
| `TASK_NOT_FOUND` | 404 | 创作任务或任务记录不存在 | 否 |
| `SCRIPT_SCHEMA_INVALID` | 422 | 剧本结构不合法 | 否 |
| `SCRIPT_DURATION_EXCEEDED` | 422 | 总时长或分镜时长不合法 | 否 |
| `COMPLIANCE_CHECK_FAILED` | 422 | 文案命中合规拦截 | 否 |
| `TASK_STATUS_CONFLICT` | 409 | 当前任务状态不允许执行该操作 | 否 |
| `IDEMPOTENCY_CONFLICT` | 409 | 幂等键冲突 | 视情况 |
| `RATE_LIMITED` | 429 | 命中限流或模型配额限制 | 是 |
| `MODEL_PROVIDER_FAILED` | 502 | 下游模型能力失败 | 是 |
| `OBJECT_STORAGE_WRITE_FAILED` | 502 | 对象存储写入失败 | 是 |
| `INTERNAL_WORKER_CALLBACK_FAILED` | 502 | 内部回调处理失败 | 是 |
| `INTERNAL_SERVER_ERROR` | 500 | 系统内部错误 | 视情况 |

### 16.2 失败处理原则
1. 所有失败响应必须包含 `trace_id`。
2. 用户可修复错误需明确字段级原因。
3. 可恢复错误需在 `error.retryable` 中给出语义。
4. 任务失败后，前端仍可通过创作详情接口和任务详情接口查询失败原因。
5. 前端是否显示“重试”按钮，应以错误码和状态为准，不应只依赖文案判断。

## 18. AI / 媒体处理能力承接与实现边界说明

### 18.1 对外承接方式
创作模块对外只通过以下结构承接 AI 与媒体处理结果：
1. `Creation`：承接创作总体状态、进度、导出结果。
2. `ShotRender`：承接分镜级渲染状态与缓存命中结果。
3. `PreviewComposition`：承接前端 Remotion Player 需要的预览态编排数据。
4. `TaskSummary` 与 SSE：承接长任务观察与阶段反馈。

### 18.2 对外不暴露的内容
1. 不暴露 Prompt 文本与推理过程。
2. 不暴露图生视频、TTS、字幕对齐、BGM Ducking 的内部参数。
3. 不暴露 FFmpeg 命令、loudnorm 参数模板、缓存路径与对象存储内部目录。
4. 不暴露队列内部结构、Redis 锁、模型端点和凭据。

### 18.3 预览与导出边界
1. 预览是前端同构渲染编排结果，面向浏览器预览体验。
2. 导出是服务端正式成片结果，面向最终 MP4 交付。
3. 两者共享同一创作语义，但不是同一资源对象，不得混写。

### 18.4 任务恢复边界
1. 页面刷新或重新进入时，前端应依赖 `creation_id` 查询详情，再以 `task_id` 恢复 SSE 观察。
2. 恢复的是“任务观察状态”，不是重新创建任务。

## 19. 命名规范、版本约定与变更边界

### 19.1 命名规范
1. URL 统一使用小写复数资源名：`creations`、`tasks`。
2. API 字段统一使用 snake_case：`creation_id`、`task_id`、`script_shot_id`、`slice_id`、`selected_slice_id`。
3. 枚举统一使用大写下划线风格：`QUEUE_ALLOCATION`。
4. 错误码统一使用大写下划线风格：`MODEL_PROVIDER_FAILED`。

### 19.2 版本约定
1. 当前统一使用 `/api/v1` 与 `/api/internal/v1`。
2. 后续新增创作能力接口时，必须先满足全局 API 版本治理要求。
3. 不得在创作模块文档中引入与全局文档冲突的字段别名。

### 19.3 兼容性敏感变更
以下变更属于高敏感接口变更，必须评审后实施：
1. `CreationStatus`、`CreationStage`、`ShotRenderStatus` 新增或修改。
2. `creation_id`、`task_id`、`script_shot_id`、`slice_id`、`selected_slice_id` 语义变化。
3. 预览载荷结构、SSE 事件结构、导出默认格式变化。
4. 内部回调字段调整导致 Worker 与网关协同协议变化。
