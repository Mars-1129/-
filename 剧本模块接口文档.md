# TikStream AI 剧本模块接口文档

## 1. 文档目标与适用范围

### 1.1 文档名称
TikStream AI——电商场景 AIGC 带货视频生成系统剧本模块接口文档

### 1.2 文档目标
本文档用于细化 TikStream AI 剧本模块的接口契约，明确前端智能剧本编辑器、NestJS Script Controller、DTO / Schema 校验层、Prompt 生成链路、模板调用链路、爆款拆解引用链路与后续创作模块之间的交互边界，作为前后端联调、后端实现、长任务协同和答辩复核的统一依据。

### 1.3 适用范围
本文档覆盖以下剧本域接口与规则：
- 快速模式剧本生成接口
- 爆款仿写剧本生成接口
- 模板模式剧本生成接口
- 剧本详情接口
- 剧本 JSON Patch 局部更新接口
- 分镜配时校验接口
- 剧本保存接口
- 模板列表 / 详情 / 应用接口
- 爆款拆解创建 / 查询接口
- 剧本域校验规则、权限规则、限流规则、错误码、示例报文
- AI 能力承接边界与 JSON Patch 白名单约束

### 1.4 文档定位
本文档是 [全局API接口文档.md](全局API接口文档.md) 在剧本域的细化分册。全局 Header、统一请求 / 响应结构、统一错误码体系、分页规则、SSE 规范、内外部接口边界均继承自全局 API 文档；本文档仅补充剧本模块特有的字段契约、生成模式、编辑规则、配时校验、合规拦截与模板 / 爆款拆解联动设计，不重复发明全局协议。

## 2. 与全局 API 规范的继承关系

### 2.1 路径与版本继承
- Base URL：`/api`
- 当前主版本：`v1`
- 剧本模块对外完整路径前缀：`/api/v1`

### 2.2 通信协议继承
- 默认响应格式：`application/json`
- 局部编辑接口：`application/json-patch+json`
- 传输协议：HTTPS

### 2.3 通用 Header 继承
| Header | 必填 | 说明 |
|---|---:|---|
| Authorization | 否 | 当前竞赛版本可选，预留 Bearer Token |
| X-Trace-Id | 否 | 调用方可传入链路追踪 ID，未传则后端生成 |
| X-Request-Id | 否 | 单次请求标识 |
| Idempotency-Key | 否 | 生成接口与爆款拆解接口建议传入 |
| Content-Type | 是 | 与接口媒体类型匹配 |

### 2.4 统一响应结构继承
剧本模块所有接口均继承 [全局API接口文档.md](全局API接口文档.md) 的响应包裹结构：
- 成功响应：`success`、`message`、`data`、`trace_id`、`timestamp`
- 失败响应：`success`、`message`、`error.code`、`error.details`、`error.retryable`、`trace_id`、`timestamp`

### 2.5 分页规范继承
模板列表接口继承全局分页规范：
- `page`
- `page_size`
- `sort_by`
- `sort_order`

### 2.6 JSON Patch 继承
剧本局部更新接口采用 RFC 6902 JSON Patch 协议，`Content-Type` 必须为 `application/json-patch+json`。每条操作至少包含：
- `op`
- `path`
- `value`（`remove` / `move` 时可省略）
- `from`（仅 `op=move` 时必填，且仅允许 `/shots/{index}` -> `/shots/{index}` 的整分镜重排）

### 2.7 本文档补充边界
本文档不直接暴露以下内部能力细节：
- Prompt 文本与元提示词模板
- Doubao 模型端点、调用凭据与配额实现细节
- 模板 `factor_json`、`schema_json` 的底层存储结构
- 爆款拆解 `strategy_json`、`factor_json`、`report_json` 的内部展开结构
- 上下文压缩策略与内部重试编排细节

## 3. 剧本域对象模型与关键枚举

### 3.1 Script 对象
Script 为一条结构化、机器可执行的剧本主对象，对应数据库 `scripts` 表。

| 字段 | 类型 | 必返 | 说明 |
|---|---|---:|---|
| script_id | string(UUID) | 是 | 剧本主键，对应 `scripts.id` |
| product_id | string(UUID) | 是 | 所属商品 ID |
| title | string | 否 | 剧本标题 |
| language | string | 否 | 输出语言，默认 `zh-CN` |
| target_audience | string | 否 | 目标受众描述 |
| video_duration | number | 是 | 剧本总时长，范围 ≤15.0 秒 |
| aspect_ratio | string | 是 | `9:16` / `16:9` |
| style_vibe | string | 是 | 风格调性 |
| generation_mode | string | 是 | 剧本生成模式 |
| template_id | string | 否 | 来源模板 ID |
| viral_video_id | string | 否 | 来源爆款拆解 ID |
| constraint_list | array | 否 | 生成约束列表 |
| raw_json | object | 是 | 原始结构化剧本快照，系统生成，用于持久化与审计；人工编辑分镜后可能与当前 shots[] 存在差异 |
| shots | ScriptShot[] | 是 | 分镜列表 |
| created_at | string(datetime) | 是 | 创建时间 |
| updated_at | string(datetime) | 是 | 更新时间 |

### 3.2 ScriptShot 对象
ScriptShot 为单个分镜对象，对应数据库 `script_shots` 表。

| 字段 | 类型 | 必返 | 说明 |
|---|---|---:|---|
| id | string(UUID) | 是 | 分镜记录主键，对应 `script_shots.id` |
| shot_id | string | 否 | 分镜业务 ID |
| shot_index | int | 是 | 分镜顺序，1-based |
| duration | number | 是 | 分镜时长，范围 `1.5~5.0` 秒 |
| scene_description_query | string | 是 | 面向素材检索的语义查询文本 |
| visual_description | string | 是 | 面向渲染与复核的可执行画面描述 |
| camera_movement | string | 是 | 运镜方式 |
| transition_type | string | 是 | 转场方式 |
| voiceover_text | string | 是 | 旁白文本 |
| subtitle_text | string | 是 | 字幕文本 |
| safe_zone_bounding_box | [number, number, number, number] | 是 | 归一化安全区坐标 `[x1, y1, x2, y2]`，各元素需位于 `[0,1]` 且满足 `x1 < x2`、`y1 < y2` |
| selected_slice_id | string | 否 | 手动指定或最终确认使用的素材切片 ID |
| render_prompt | string | 否 | 创作侧图生视频或补帧承接提示 |
| local_factor_patch | object | 否 | 局部风格 / 因子替换结果 |
| compliance_status | string | 否 | `PENDING` / `PASSED` / `REJECTED` |
| created_at | string(datetime) | 是 | 创建时间 |
| updated_at | string(datetime) | 是 | 更新时间 |

### 3.3 Template 对象
Template 为模板市场中的策略模板对象，对应数据库 `templates` 表。对外接口不暴露完整 `factor_json` 与 `schema_json` 内部结构，只返回可供选择与应用的摘要字段。

| 字段 | 类型 | 必返 | 说明 |
|---|---|---:|---|
| template_id | string(UUID) | 是 | 模板主键 |
| product_id | string(UUID) | 否 | 可选商品关联 |
| name | string | 是 | 模板名称 |
| category | string | 是 | 模板分类 |
| strategy_summary | string | 是 | 模板策略摘要 |
| status | string | 是 | 模板状态 |
| created_at | string(datetime) | 是 | 创建时间 |
| updated_at | string(datetime) | 是 | 更新时间 |

### 3.4 ViralVideoAnalysis 对象
ViralVideoAnalysis 为爆款视频结构化拆解结果对象，对应数据库 `viral_video_analyses` 表。对外接口只暴露引用级字段与可被剧本模块消费的摘要结果，不返回原视频本体，也不强制展开内部策略树。

| 字段 | 类型 | 必返 | 说明 |
|---|---|---:|---|
| analysis_id | string(UUID) | 是 | 拆解记录主键，对应数据库 `viral_video_analyses.id` |
| product_id | string(UUID) | 否 | 可选商品关联 |
| source_platform | string | 是 | 来源平台 |
| source_url | string | 是 | 公开来源链接 |
| external_video_id | string | 是 | 平台视频 ID |
| title | string | 否 | 视频标题 |
| hook_type | string | 否 | Hook 手法摘要 |
| declared_public_source | boolean | 是 | 是否公开来源声明 |
| created_at | string(datetime) | 是 | 创建时间 |
| updated_at | string(datetime) | 是 | 更新时间 |

### 3.5 关键枚举

#### 3.5.1 ScriptGenerationMode
继承 [数据库设计文档.md](数据库设计文档.md) 中的枚举定义：
- `PROMPT_DRIVEN`
- `VIRAL_REWRITE`
- `TEMPLATE_DRIVEN`

### 3.5.2 TemplateStatus
- `ACTIVE`
- `INACTIVE`
- `ARCHIVED`

### 3.5.3 AspectRatio
- `9:16`
- `16:9`

### 3.5.4 CameraMovement
- `Static`
- `Dolly_In_Fast`
- `Dolly_Out`
- `Pan_Left`
- `Tilt_Up`

### 3.5.5 TransitionType
- `None`
- `Fade_In`
- `Dissolve`
- `Wipe`

### 3.5.6 ComplianceStatus
- `PENDING`
- `PASSED`
- `REJECTED`

### 3.6 唯一键与定位约束
1. `script_id` 是剧本域对外主定位键。
2. `script_id + shot_index` 是分镜稳定定位组合键，必须全局沿用，不得替换为其他别名。
3. `selected_slice_id` 是剧本分镜承接素材切片绑定关系的标准字段。
4. 模板与爆款拆解引用时，`template_id`、`analysis_id` 仅作为引用键，不等于对外暴露其底层完整策略数据。

### 4.3 页面动作映射
- `/scripts` 页面（通过 `?productId=` 绑定商品上下文）快速模式生成：调用 `POST /api/v1/scripts/generate/quick`
- `/scripts` 页面爆款仿写：先依赖爆款拆解结果，再调用 `POST /api/v1/scripts/generate/viral-rewrite`
- `/templates` 页面选择模板并套用：调用 `POST /api/v1/templates/{template_id}/apply` 或 `POST /api/v1/scripts/generate/template`
- `/scripts` 页面打开已有剧本：调用 `GET /api/v1/scripts/{script_id}`
- `/scripts` 页面拖拽重排、改台词、改素材、改局部因子：调用 `PATCH /api/v1/scripts/{script_id}`
- `/scripts` 页面实时检查台词超时：调用 `POST /api/v1/scripts/{script_id}/validate-timing`
- `/scripts` 页面点击保存：调用 `POST /api/v1/scripts/{script_id}/save`

## 5. 快速模式剧本生成接口

### 5.1 接口说明
- 方法：`POST`
- 路径：`/api/v1/scripts/generate/quick`
- 作用：根据商品信息、卖点、受众和风格约束直接生成结构化分镜剧本。
- 权限：当前商品上下文内可写
- 限流：共享剧本生成类接口的 Doubao 文本模型令牌桶
- 幂等：建议传入 `Idempotency-Key`

### 5.2 请求字段
| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| product_id | string(UUID) | 是 | 商品 ID |
| title | string | 否 | 剧本标题 |
| language | string | 否 | 输出语言，默认 `zh-CN` |
| selling_points | string[] | 是 | 商品卖点列表 |
| target_audience | string | 否 | 目标人群 |
| style_vibe | string | 是 | 风格调性 |
| aspect_ratio | string | 是 | `9:16` / `16:9` |
| constraint_list | string[] | 否 | 额外约束，如总时长、避免词汇、风格禁用项 |

### 5.3 校验规则
1. `product_id` 必填且必须存在。
2. `selling_points` 至少包含 1 项，建议不超过 10 项。
3. `aspect_ratio` 仅允许 `9:16` 或 `16:9`。
4. 模型输出必须满足剧本 Schema 强校验。
5. 总时长必须小于等于 15 秒，单分镜时长必须位于 `1.5~5.0` 秒。
6. 每个分镜必须包含 `duration`、`scene_description_query`、`visual_description`、`camera_movement`、`transition_type`、`voiceover_text`、`subtitle_text`、`safe_zone_bounding_box`。

### 5.4 成功响应字段
| 字段 | 类型 | 说明 |
|---|---|---|
| script_id | string(UUID) | 新生成剧本 ID |
| product_id | string(UUID) | 商品 ID |
| video_duration | number | 总时长 |
| aspect_ratio | string | 画幅 |
| style_vibe | string | 风格调性 |
| generation_mode | string | `PROMPT_DRIVEN` |
| shots | array | 结构化分镜列表 |
| created_at | string(datetime) | 创建时间 |

### 5.6 失败场景
- `PRODUCT_ID_REQUIRED`：缺失商品 ID

## 6. 爆款仿写剧本生成接口

### 6.1 接口说明
- 方法：`POST`
- 路径：`/api/v1/scripts/generate/viral-rewrite`
- 作用：基于公开来源爆款视频的结构化拆解结果，为新商品生成同类叙事结构剧本。
- 权限：当前商品上下文内可写
- 限流：共享剧本生成类接口限流
- 幂等：建议传入 `Idempotency-Key`

### 6.2 设计边界
1. 本接口消费的是爆款视频的结构化拆解结果，而不是原视频媒体本体。
2. 仅允许使用已声明公开来源的拆解记录。
3. 返回结果只包含新生成剧本，不复刻、不回传原视频片段。
4. 文档与实现中必须保留来源声明边界。

### 6.3 请求字段
| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| product_id | string(UUID) | 是 | 商品 ID |
| viral_video_id | string | 是 | 爆款拆解记录 ID，对应 `analysis_id` |
| title | string | 否 | 剧本标题 |
| language | string | 否 | 输出语言 |
| style_vibe | string | 是 | 风格调性 |
| aspect_ratio | string | 是 | `9:16` / `16:9` |
| selling_points | string[] | 否 | 可选卖点补充，用于填充新商品表达 |
| target_audience | string | 否 | 目标受众 |
| constraint_list | string[] | 否 | 额外约束 |

### 6.4 校验规则
1. `viral_video_id` 必须存在，且拆解记录可用。
2. 拆解记录必须为公开来源声明数据。
3. 不允许将原视频媒体信息直接返回给前端。
4. 返回结果仍需通过剧本 Schema 强校验与总时长校验。

## 11. 剧本保存接口

### 11.1 接口说明
- 方法：`POST`
- 路径：`/api/v1/scripts/{script_id}/save`
- 作用：对已编辑剧本执行最终校验并持久化保存，作为后续创作模块的正式输入。
- 权限：当前商品上下文内可写

### 11.2 保存前置条件
1. 剧本存在。
2. 剧本至少包含 1 个分镜。
3. 所有分镜字段完整。
4. 所有分镜顺序连续且 `shot_index` 唯一。
5. 服务端会在保存时基于当前剧本状态重新执行分镜配时校验与合规校验，前端预校验结果不能替代最终保存校验。

### 11.3 阻断条件
以下情况任一命中即阻断保存：
- 总时长超过 15 秒
- 单分镜时长不在 `1.5~5.0` 秒范围内
- 任一分镜旁白配时超出其分配时长
- 缺失关键字段
- 命中合规词法拦截规则
- JSON Patch 后形成非法结构
- `script_id + shot_index` 唯一性被破坏

### 11.4 请求字段
本接口可不要求复杂请求体；默认以当前服务端已暂存的剧本状态作为保存对象。可选补充：

| save_message | string | 否 | 本次保存说明 |
| force_revalidate | boolean | 否 | 是否强制重新执行全部校验，默认 `true` |

## 13. 校验规则、权限与限流规则

### 13.1 剧本字段校验规则汇总
1. `video_duration` 必须在 ≤15.0 秒范围内。
2. `shots` 不允许为空。
3. 单分镜 `duration` 必须在 `1.5~5.0` 秒范围内。
4. `shot_index` 必须为正整数，且在同一 `script_id` 下唯一。
5. `safe_zone_bounding_box` 必须为归一化四元坐标数组 `[x1, y1, x2, y2]`，各分量值域 0~1。
6. `scene_description_query`、`visual_description`、`voiceover_text`、`subtitle_text` 不允许缺失。
7. `camera_movement` 与 `transition_type` 必须取自既定枚举。
8. `selected_slice_id` 若传入，必须与素材切片域的 `slice_id` 命名口径一致。

### 13.2 配时与总时长规则
1. 总时长大于 15 秒时阻断保存。
2. 分镜配时以快速估算结果为准，若 `estimated_duration > duration` 则阻断。
3. 分镜拖拽重排后必须重新校验总时长和分镜顺序连续性。

### 13.3 合规校验规则
1. 对绝对化营销表达执行词法拦截。
2. 命中合规风险时返回 `COMPLIANCE_CHECK_FAILED`。
3. `compliance_status` 为系统写入字段，不允许客户端直接改写。

### 13.4 权限规则
- 当前竞赛版本采用轻量化单商家口径。
- 用户仅允许在当前商品上下文内查询和编辑剧本。
- 模板应用与爆款拆解引用也必须受商品上下文约束。
- 内部策略结构、模型细节与服务端校验状态不对前端直接开放编辑权限。

### 13.5 限流规则
| 接口类型 | 规则 |
|---|---|
| 剧本生成类接口 | 共享 Doubao 文本模型 Token Bucket，控制在 80 RPM 以内 |
| 剧本详情接口 | 常规读接口限流 |
| JSON Patch 编辑接口 | 按用户与剧本维度限频，防止高频拖拽风暴 |
| 配时校验接口 | 按用户与剧本维度限频，允许较高频但需要防抖 |
| 保存接口 | 按剧本维度限频，避免重复保存风暴 |
| 爆款拆解创建接口 | 受外部解析与模型调用资源约束 |

### 13.6 幂等规则
- 三类生成接口建议传入 `Idempotency-Key`，避免重复创建多条相同剧本。
- 爆款拆解创建接口建议以 `source_platform + external_video_id` 作为重复提交判定基础之一。
- 保存接口在重复请求时不应生成重复记录，只更新同一剧本状态。

## 14. 错误码与失败处理规范

### 14.1 复用的全局错误码
| 错误码 | HTTP 状态码 | 含义 | 可重试 |
|---|---:|---|---:|
| PRODUCT_ID_REQUIRED | 400 | 缺失 `product_id` | 否 |
| INVALID_REQUEST | 400 | 请求体、Patch 协议或字段格式错误 | 否 |
| SCRIPT_NOT_FOUND | 404 | 剧本不存在 | 否 |
| TEMPLATE_NOT_FOUND | 404 | 模板不存在 | 否 |
| VIRAL_VIDEO_ANALYSIS_NOT_FOUND | 404 | 爆款拆解记录不存在 | 否 |
| SCRIPT_SCHEMA_INVALID | 422 | 剧本结构字段不完整或不合法 | 否 |
| SCRIPT_DURATION_EXCEEDED | 422 | 总时长或分镜时长、配时不合法 | 否 |
| COMPLIANCE_CHECK_FAILED | 422 | 命中合规词法拦截规则 | 否 |
| RATE_LIMITED | 429 | 命中模型配额或接口限流 | 是 |
| MODEL_PROVIDER_FAILED | 502 | 外部模型调用失败 | 是 |
| INTERNAL_SERVER_ERROR | 500 | 未分类内部错误 | 视情况 |

### 14.2 剧本域失败处理原则
1. 用户可修复错误必须返回字段级问题说明。
2. 模型侧可恢复失败返回 `retryable=true`。
3. 所有失败都必须返回 `trace_id`，便于联调和答辩时追踪。
4. 配时失败与合规失败必须是强阻断，而不是仅提示。

## 16. AI 能力承接与 JSON Patch 约束说明

### 16.1 AI 能力承接说明
剧本模块当前不向前端开放任何“模型直连接口”，而是通过业务接口承接 AI 结果：
1. 快速模式、爆款仿写、模板模式均通过剧本生成接口返回结构化剧本字段。
2. 爆款拆解能力通过 `POST /api/v1/viral-video-analyses` 与 `GET /api/v1/viral-video-analyses/{analysis_id}` 暴露结构化分析承接入口。
3. 模板能力通过模板列表 / 详情 / 应用接口暴露业务语义，不暴露内部策略 JSON 细节。
4. 配时校验对外暴露的是结果与建议，不暴露内部音节推演公式实现细节。
5. 合规拦截对外暴露的是阻断结论与整改建议，不暴露完整规则库。

### 16.2 安全边界说明
- 不暴露 Prompt 文本、模型端点、API Key、上下文压缩策略与资源地址。
- 不暴露模板底层 `factor_json` / `schema_json`。
- 不暴露爆款拆解的完整内部策略结构与模型调用细节。
- 不返回原爆款视频媒体内容，只返回结构化分析结果与来源声明。

### 16.3 JSON Patch 约束说明
1. 仅允许对白名单路径执行 `add` / `replace` / `remove`。
2. 编辑后若影响下游重渲染或创作任务，应由创作模块后续链路承接，不在本分册扩展新的任务协议。
3. 对分镜数组的新增 / 删除必须保持 `shot_index` 最终可重排为连续序列。
4. 任何破坏结构化 Schema 的 Patch 都必须被后端拒绝。
