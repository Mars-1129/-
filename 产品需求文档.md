# TikStream AI 产品需求文档（PRD）

## 1. 文档概述

### 1.1 文档名称
TikStream AI——电商场景 AIGC 带货视频生成系统产品需求文档

### 1.2 文档目标
本文档用于定义 TikStream AI 的完整产品需求，明确系统的业务目标、用户场景、功能模块、交互逻辑、异常处理、优先级划分与验收口径，为设计、开发、测试和答辩提供统一依据。

### 1.3 文档范围
本文档覆盖以下产品能力：
- 素材管理与结构化处理
- 智能剧本生成与编辑
- 创作与智能剪辑
- 数据分析看板与反馈优化
- 页面工作台、任务管理、模板市场等配套能力

### 1.4 文档定位
本文档采用“竞赛 MVP 正式交付版 + 后续商业化演进兼顾”的口径。正文优先保证竞赛交付能力完整、工程逻辑可实现，同时在文末保留扩展规划。

## 2. 产品概述与价值主张

### 2.1 产品定位
TikStream AI 是面向 TikTok Shop 等国际电商平台的一站式 AIGC 带货视频生成系统。用户只需上传商品图片、视频，或输入商品核心卖点，即可自动生成符合平台规范、具备转化导向的 15 秒竖屏带货短视频。

### 2.2 核心价值主张
1. 降低短视频生产门槛，让没有专业团队的商家也能完成视频创作。
2. 缩短从素材到成片的生产周期，提高营销素材产出效率。
3. 支持模板化、结构化、可复用的内容生成方式，降低重复劳动。
4. 提供从生成到分析的闭环能力，支撑后续优化与增长策略调整。

### 2.3 产品目标
- 支撑竞赛版本从素材处理到成片导出的完整主流程。
- 通过结构化设计兼顾业务可信度与工程落地能力。
- 为后续扩展至商业化场景留下接口和架构空间。

## 3. 用户角色与核心场景

### 3.1 用户角色

#### 角色一：跨境电商中小商家
- 典型特征：没有摄制团队，只有商品主图、白底图或少量视频。
- 核心诉求：快速生成可上架、可投流的视频素材。

#### 角色二：品牌跨境运营人员
- 典型特征：需要频繁产出多版本素材进行测试。
- 核心诉求：批量生成不同风格、多语言、多版本视频。

#### 角色三：内容营销与剪辑团队
- 典型特征：有剪辑经验，但缺乏系统化方法论沉淀。
- 核心诉求：复用爆款结构、保留高转化策略模板。

#### 角色四：工厂/档口卖家
- 典型特征：拥有横屏实拍长视频，但不会做平台适配剪辑。
- 核心诉求：自动切片、重组、裁切并生成竖屏短视频。

### 3.2 典型场景
1. 商家上传商品图和卖点文案，一键生成带货视频。
2. 商家上传横屏素材，系统自动切片并适配 9:16 竖屏。
3. 运营选择爆款仿写模式，复用某条高转化视频的结构。
4. 运营手动修改某个分镜文案后，系统只重渲染该分镜。
5. 商家在看板中查看 A/B 两版视频的预测表现并进行优化。

## 4. 端到端业务流程

### 4.1 主流程
```text
素材上传
  → 素材切片与打标
  → 商品级资产入库
  → 选择剧本生成模式
  → 输出结构化分镜剧本
  → 分镜编辑与配时校验
  → 一键成片任务创建
  → 素材匹配 / 图生视频兜底
  → 配音 / 字幕 / BGM / 拼接
  → 视频导出
  → 分析看板查看效果与优化建议（基于 Mock / 预计算数据）
```

### 4.2 用户任务流说明
1. 资产摄入：用户上传图片或视频，系统完成结构化处理。
2. 模式选择：用户选择快速模式、爆款仿写或模板生成。
3. 分镜干预：用户对分镜文案、素材、时长、运镜等进行调整。
4. 即时预览：用户在前端工作台实时查看效果。
5. 一键成片：系统异步处理创作任务并输出导出链接。
6. 效果分析：用户在分析看板中查看 Mock / 预计算的预测表现并决定是否继续优化。

## 5. 功能架构总览

系统功能分为四个核心业务模块与两个配套模块：

1. 素材模块（人员 A）
2. 剧本模块（人员 B）
3. 创作模块（人员 A）
4. 分析看板模块（人员 B）
5. 任务管理模块（公共）
6. 模板市场模块（公共）

## 6. 模块需求详述

### 6.1 素材模块（人员 A）

#### 6.1.1 模块目标
将用户上传的商品素材转化为“可被脚本和创作模块直接消费”的结构化资产。

#### 6.1.2 功能需求

##### FR-1 素材上传（P0）
- 支持上传 MP4、JPG、PNG、WebP 文件。
- 视频最大 200MB，图片最大 10MB。
- 前端采用分片上传，支持断点续传。
- 上传时必须绑定商品 product_id。
- 上传完成后生成缩略图。

##### FR-2 素材切片与 Dense Caption（P0）
- 视频上传完成后触发切片任务。
- 切片时长默认控制在 1.5s 到 4.0s 区间。
- 单镜头大于 5.0s 时强制等距切分为 3.0s 级别切片。
- 小于 1.5s 的碎片直接舍弃，不进入下游创作链路。
- 对关键帧进行语义描述生成 Dense Caption。
- 切片元数据持久化。

##### FR-3 多模态向量化与检索（P1）
- 对切片内容生成向量表示。
- 支持按文本语义检索相关切片。
- 支持按商品 ID、时长等维度做结构化过滤。
- 支持关键字模糊搜索作为兜底方案。

##### FR-4 9:16 自适应裁切（P2）
- 对横屏素材做商品主体区域识别。
- 在预览和导出阶段保持主体处于安全区域。

##### FR-5 物理音效提取（P2）
- 从原始素材中提取可复用物理交互音效。
- 供后续创作模块叠加使用。

#### 6.1.3 业务规则
- 所有上传素材必须归属到明确商品。
- 碎片过短的切片不进入创作链路。
- 素材状态应支持处理中、成功、失败等标记。

#### 6.1.4 异常场景
- 上传中断：支持断点恢复。
- 上传格式错误：立即拒绝并提示用户。
- 切片失败：素材标记失败，并允许重新处理。
- 打标限流：进入队列重试或降级模式。

### 6.2 剧本模块（人员 B）

#### 6.2.1 模块目标
根据商品信息、模板或参考视频生成结构化、机器可执行的短视频分镜剧本。

#### 6.2.2 功能需求

##### FR-6 快速模式剧本生成（P0）
- 用户输入商品卖点、受众、风格等信息。
- 系统生成 15 秒内分镜化短视频脚本。
- 输出内容必须结构化，可直接用于创作模块。

##### FR-7 爆款仿写模式（P1）
- 用户指定参考视频或爆款模板。
- 系统抽取其结构特征并应用于新商品。

##### FR-8 灵感模板模式（P1）
- 用户在模板市场中选择模板。
- 系统将模板结构与商品信息融合输出定制剧本。

##### FR-9 文案偏好对齐（P2）
- 系统通过 Winner/Loser 样例约束文案风格。
- 强调高转化表达方式，避免说明书式文本。

##### FR-10 剧本 Schema 强校验（P0）
- 输出必须包含分镜时长、画面描述、运镜、转场、旁白、字幕、安全区坐标等字段。
- `safe_zone_bounding_box` 必须为归一化四元坐标 `[x1, y1, x2, y2]`，所有坐标值域 `0~1`，用于前端安全区预览与字幕定位。
- 禁止输出不可执行、歧义化的文学表达。

##### FR-11 分镜编辑与配时校验（P1）
- 支持通过 Prompt 微调入口对整条剧本做快速重生成。
- 支持手动改写台词、增删分镜、替换素材、调整分镜顺序与局部因子。
- 系统实时校验台词长度与分镜时长是否匹配。
- 超出时长应阻断保存并给出修改建议。

##### FR-12 合规词法拦截（P1）
- 基础绝对化用语、禁止性广告表达进行识别与阻断（P1）。
- 高级语境化合规识别与细粒度策略拦截（P2）。

#### 6.2.3 业务规则
- 视频总时长不超过 15 秒。
- 单分镜时长需满足 1.5s~5.0s 约束。
- 每个分镜必须绑定明确的视觉与语音字段。

#### 6.2.4 异常场景
- 大模型响应超时：提示重试并支持降级。
- 剧本字段缺失：校验失败，不允许进入创作流程。
- 分镜超时长：阻断保存。
- 模板缺失：提示模板无效或已下线。

### 6.3 创作模块（人员 A）

#### 6.3.1 模块目标
将结构化脚本与素材资产转换为实际可导出的带货视频成品。

#### 6.3.2 功能需求

##### FR-13 一键成片（P0）
- 用户点击后创建异步任务，系统返回 `creation_id`（创作记录主键）与 `task_id`（任务追踪主键），前者用于接口操作，后者用于前端进度观察与 SSE 订阅。
- 系统基于分镜逐一匹配素材并执行合成。

##### FR-14 图生视频兜底（P0/P1）
- 当切片检索不到合适素材时，系统自动触发图生视频补足链路。

##### FR-15 四轨合成（P0/P1）
- 视频轨：优先使用缓存切片或匹配素材。
- 旁白轨：根据分镜文案生成配音。
- 字幕轨：基于旁白时间戳生成字幕。
- BGM 轨：按风格召回音乐并执行音量避让。

##### FR-16 分镜级局部重渲染（P1）
- 用户修改单个分镜后，只对该分镜重新生成与合成。
- 未变更分镜复用缓存结果。

##### FR-17 即时预览（P1/P2）
- 前端提供高还原度的实时预览能力。
- 预览与最终导出保持尽可能一致。

##### FR-18 最终导出（P0）
- 默认输出 1080×1920，30fps，H.264 标准视频。
- 支持竖版 9:16 与横版 16:9 等多画幅导出，适配不同分发渠道。
- 支持导出链接返回与下载查看。

##### FR-19 响度标准化（P2）
- 保证最终视频音频响度满足统一标准。

#### 6.3.3 业务规则
- 成片时长默认不超过 15 秒。
- 分镜缓存命中时不重复渲染。
- 导出结果必须落地到对象存储。

#### 6.3.4 异常场景
- 素材检索失败：走图生视频兜底链路。
- 图生视频限流：任务进入延迟重试。
- 配音失败：任务标记失败并返回原因。
- 渲染失败：保留 trace 信息供排查。

### 6.4 分析看板模块（人员 B）

#### 6.4.1 模块目标
利用 Mock 数据构建内容效果分析闭环，支持策略评估与下一轮内容优化。

#### 6.4.2 功能需求

##### FR-20 多因子归因热力图（P2）
- 展示视觉风格与叙事策略组合对效果的贡献。

##### FR-21 视听留存桑基图（P2）
- 展示 BGM、视觉风格与完播率区间之间的关系。

##### FR-22 分镜留存曲线（P1）
- 展示视频时间轴上的用户留存衰减。
- 标记分镜切换点与下滑拐点。

##### FR-23 A/B 对比（P2）
- 支持同一商品不同视频版本的预测表现比较。

##### FR-24 一键自愈（P2）
- 根据看板诊断结果回写创作链路，优化受影响分镜。

#### 6.4.3 业务规则
- 看板以 Mock 数据和预计算结果为基础。
- 所有指标需标注其为预测值或模拟值。

#### 6.4.4 异常场景
- 预计算数据缺失：返回兜底提示。
- A/B 数据不完整：只显示可用维度。

### 6.5 任务管理模块（公共）

#### 功能需求
- 查看任务列表、状态、进度和错误信息。
- 支持任务探活、失败重试和主动取消。
- 支持刷新页面后恢复长任务状态。

### 6.6 模板市场模块（公共）

#### 功能需求
- 展示预置模板分类和示例。
- 支持模板选择、预览和一键套用。

## 7. 页面与交互需求

### 7.1 页面清单
1. /materials：素材管理工作台
2. /scripts：智能剧本编辑器（通过 query 参数 `?productId=` 绑定当前商品上下文，选择具体剧本后以 `script_id` 为核心标识）
3. /create：创作与预览工作台
4. /tasks：任务历史页
5. /analytics：分析看板页
6. /templates：模板市场页

### 7.2 核心交互要求
- 素材上传需支持拖拽、分片与进度反馈。
- 分镜编辑器需支持拖拽排序、字段编辑、实时校验、快捷键操作与时间轴协同编辑。
- 分镜级修改后应支持局部刷新预览，避免每次都触发整片重渲染。
- 预览区需展示 9:16 安全区域，并支持不同画幅下的预览切换。
- 长任务需通过 SSE 实时反馈进度，并支持断点恢复、失败重试与清晰失败原因提示。
- 导出按钮需明确区分“预览”和“正式导出”，并支持多分辨率、多画幅导出切换。
- 当前前端统一采用 shadcn/ui 作为通用交互组件库，采用 Tailwind CSS 作为响应式布局与样式体系；视频预览由 Remotion 4 Player 承担，分析看板图表由 ECharts 5 承担。
- 分镜卡片、表单控件、弹窗、抽屉、下拉选择、滑块、标签、按钮等通用工作台交互，统一基于 shadcn/ui 封装，避免页面间交互风格不一致。
- 移动端商家需支持轻量查看任务进度、预览结果与进行有限字段级编辑，复杂时间轴编辑以桌面端为主。

## 8. 权限规则与状态流转

### 8.1 权限规则
竞赛版本采用轻量化单商家口径，不设计复杂组织权限。用户在当前商品上下文内可访问与编辑相关素材、剧本、创作任务与分析结果。

### 8.2 状态流转

#### 素材状态
PENDING → PROCESSING → COMPLETED / FAILED

#### 创作任务状态
QUEUE_ALLOCATION → ASSET_MATCHING → AI_VIDEO_GENERATING → TTS_GENERATING → FFMPEG_STITCHING → LOUDNORM_COMPLIANCE → FINISHED / FAILED

## 9. 异常场景与失败处理

### 9.1 用户侧异常
- 文件格式错误
- 文件体积超限
- 台词长度超出分镜容量
- 网络刷新导致长任务中断

### 9.2 系统侧异常
- API 限流
- 向量检索失败
- 图生视频并发占满
- 渲染失败
- 对象存储写入失败

### 9.3 处理原则
- 优先返回可理解、可追踪的错误信息。
- 对可恢复任务进行自动重试或延迟重试。
- 对不可恢复问题保留 trace 与失败原因。

## 10. 非功能需求

### 10.1 性能要求
- 主流程可在竞赛环境下可运行、可演示。
- 看板加载应保持较快响应。
- 分镜级改动优先走局部渲染，避免全片重算。

### 10.2 稳定性要求
- 具备限流、并发控制、重试和降级能力。
- 长任务在页面刷新后可恢复观察状态。

### 10.3 可维护性要求
- 模块边界清晰。
- 类型契约统一。
- 文档与实现保持一致。

### 10.4 可扩展性要求
- 保留商业化扩展到更多模板、语言、模型和真实数据回流的空间。

## 11. 需求优先级汇总

### P0
- 素材上传
- 素材切片与 Dense Caption
- 图生视频兜底（基础链路）
- 剧本生成
- 分镜结构化
- 一键成片
- 基础音视频轨合成（视频轨+旁白轨）
- 导出
- 基础任务追踪

### P1
- 向量检索
- 模板、爆款仿写
- 分镜局部编辑
- 四轨合成（BGM轨+字幕轨完整版）
- 图生视频增强兜底
- 即时预览
- 任务恢复
- 基础看板（分镜留存曲线）

### P2
- 高还原度即时预览（与最终导出画质一致）
- A/B 对比
- 一键自愈
- 多因子归因
- 视听留存桑基图
- 响度标准化
- CI/CD 与可观测性配套展示

## 12. 验收标准

1. 用户可以围绕一个商品完成从素材上传到导出视频的全流程操作。
2. 剧本模块输出的结果为结构化、可执行的分镜数据。
3. 创作模块能够完成至少一条视频的完整生成、预览与导出。
4. 长任务具备可追踪状态、失败信息与恢复观察能力。
5. 看板模块可展示至少一套完整 Mock 分析结果。
6. 各模块职责、优先级和边界与立项书和架构文档保持一致。

## 13. 后续演进规划

1. 引入更丰富的模板市场与行业策略库。
